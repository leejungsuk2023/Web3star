# 📱 Web3Star 앱 개발 PRD (For MVP)

## 1. 프로젝트 개요

- **플랫폼:** iOS / Android (React Native Expo)
- **테마:** **Dark Theme** (배경: `#000000`, 포인트: `#3B82F6` or Logo Color)
- **핵심 기능:** 4시간 채굴 타이머, 광고 시청 후 포인트 적립 (Get More Point), 친구 초대
- **웹 홈페이지:** 별도 랜딩 페이지 필요 (섹션 7 참조)

---

## 2. 앱 구조도 (User Flow)

피그마에서 우선 **이 5개 화면**만 작업합니다.

1. **Splash (로딩)**: 로고 애니메이션
2. **Auth (인증)**: 로그인 / 회원가입 (이메일, 비번, 추천인 코드 입력)
3. **Home (메인)**: 채굴 버튼, 타이머, 광고 슬롯 (Get More Point)
4. **Leaderboard (랭킹)**: 1~100위 리스트
5. **Profile (내 정보)**: 내 포인트, 초대 코드, 공지사항/백서 링크 메뉴

---

## 3. 화면별 상세 기획 (Figma & Cursor 기준)

### 3.1 Auth Screen (로그인/회원가입)

**[Figma UI]**

- 배경: 검정 (`#000000`)
- 입력 필드: 흰색/밝은 톤 인풋 박스
- **입력 필드**
  - Email
  - Password
  - Nickname
  - Referral Code (optional)
- **버튼**
  - `Btn_SignUp`
  - `Btn_Login`

**[Cursor Logic]**

- Supabase Auth 연동
- 회원가입 시:
  - `users` 테이블에 레코드 생성
  - `invite_code`: `uuid` 앞 6자리로 자동 생성
- 추천인 코드 입력 시:
  - `referrals` 테이블(또는 `users.referred_by`)에 추천 관계 저장

---

### 3.2 Home Screen (메인 / 핵심 화면)

**[Figma UI]**

- **상단 헤더 (`Header_Main`)**
  - `Txt_Nickname` (내 닉네임)
  - `Txt_Point` (현재 포인트, 크게)
  - `Ic_Notification`
- **중앙 메인 액션**
  - `Btn_StartMining` : 거대한 W 로고 버튼 (SVG 형식 필요)
    - **채굴 가능 상태 (4시간 경과 후)**: 회색 또는 꺼진 상태
    - **채굴 중 (광고 시청 완료 후)**: 빛나는 효과(Glow, Gradient 등)로 활성화 표시
    - 버튼 클릭 → 광고 시청 → 광고 종료 시 활성화 상태로 변경
- **중앙 하단 타이머**
  - `Txt_Timer` : 형식 `03:59:59`
- **하단 포인트 획득 패널 (`Panel_GetMorePoint`)**
  - 제목: **"Get More Point"** (용어 통일 — "Ad Boost" 사용 금지)
  - 작은 원형 슬롯 5개
    - `Slot_Ad_1` ~ `Slot_Ad_5`
    - **미시청 상태**: 기본 색상 (회색 등)
    - **시청 완료 상태**: 활성화된 색상으로 변경 (각 슬롯마다 다른 색상 가능)
    - 시청한 슬롯은 "봤다"는 표시가 명확히 보이도록 처리
  - **진행률 표시**: 5개 중 몇 개를 시청했는지 시각적으로 표시

**[Cursor Logic]**

- **채굴 버튼 (`Btn_StartMining`) — W 로고 버튼**
  - W 로고 자체가 채굴 트리거 버튼 (로고를 직접 탭하여 채굴 시작)
  - 클릭 → 전면 광고 노출 (Interstitial Ad)
  - 광고 시청 완료 후 (광고 끝나면 X 버튼 클릭하여 닫기):
    - 포인트 `+10`
    - 타이머 4시간 시작
    - `last_mined_at` 저장
    - 로고 버튼을 **빛나는 활성화 상태**로 변경 (Glow 효과 등)
- **광고 슬롯 (`Slot_Ad_*`) - "Get More Point" 패널**
  - 광고를 시청하면 **추가 포인트**를 받는 개념 (부스트가 아닌 포인트 적립)
  - 클릭 → 보상형 광고 (Rewarded Ad)
  - **포인트 보상 체계 (퍼센트 보너스)**:
    - 5개 슬롯 전부 시청 시: 채굴 포인트의 **5% 추가 보너스** 지급
    - 부분 시청 시: 시청한 슬롯 수에 비례하여 보너스 (예: 3개 시청 = 3% 보너스)
    - 해당 슬롯 상태 "활성"으로 색상 변경
- **타이머 유지 로직**
  - 앱 재실행 시:
    - `last_mined_at`와 현재 시간 비교
    - 남은 시간 계산 후 `Txt_Timer`에 표시
    - 4시간이 지난 경우:
      - 다시 채굴 가능 상태로 UI 변경 (로고 버튼 회색/꺼진 상태로 변경)
      - 광고 슬롯은 리셋되지 않고 유지 (또는 리셋 정책에 따라 결정)

## 4. 데이터베이스 스키마 (Supabase)

```sql
-- Users Table
create table public.users (
  id uuid references auth.users not null primary key,
  email text,
  nickname text,
  point int8 default 0,
  invite_code text unique,
  referred_by text, -- 나를 초대한 사람 코드
  last_mined_at timestamptz, -- 마지막 채굴 시간
  ad_slots_viewed jsonb default '[]'::jsonb, -- 시청한 광고 슬롯 번호 배열 [1,2,3] 등
  created_at timestamptz default now()
);

-- Mining Logs (History)
create table public.mining_logs (
  id bigint generated by default as identity primary key,
  user_id uuid references public.users,
  amount int,
  type text, -- 'MINING' or 'AD_POINT' or 'REFERRAL' or 'BONUS'
  slot_number int, -- 광고 슬롯 번호 (1~5), MINING일 경우 null
  created_at timestamptz default now()
);
```

## 5. Figma 작업 가이드 (Cursor 친화형)

1. **레이어 이름 규칙**
   - 버튼: `Btn_StartMining`, `Btn_GetMorePoint`, `Btn_Login` 처럼 **영어 + PascalCase**로 작성
   - 텍스트: `Txt_Timer`, `Txt_Point`, `Txt_Nickname` 처럼 역할이 드러나게 작성
2. **컴포넌트화**
   - 하단 탭바(`TabBar_Main`), 상단 헤더(`Header_Main`)는 컴포넌트로 만들어 모든 화면에서 재사용
   - 버튼 스타일도 `Component_ButtonPrimary`, `Component_ButtonSecondary` 등으로 정의
3. **이미지/아이콘 Export**
   - **로고 버튼 (W 로고)**: 반드시 **SVG 형식**으로 제공 (JPG 불가)
   - 메달 아이콘, 특별한 일러스트는 1x / 2x / 3x 해상도로 Export
   - 포맷: PNG 또는 SVG (로고는 SVG 필수)
4. **UI/UX 품질**
   - 홈 화면을 포함한 모든 화면을 멋있고 세련되게 디자인
   - 로고 버튼의 활성화/비활성화 상태가 명확히 구분되도록 처리
